Strategy summary

Eine headless Playwright-e2e-Test-Infrastruktur fuer die statische Web-App web/, plus ein Build-Zeit-Versionsstempel in der Fusszeile. Die Richtung: eine einzige Abhaengigkeit (@playwright/test) deckt Runner, Browser, Server-Lifecycle, Upload und Auto-Retry-Assertions ab — kein eigener Server-Tanz, keine festen Sleeps.

  • Produktivcode zuerst, Specs danach.

    Test-Seam, ECharts-Vendoring und Stempel (Tasks 1–4) landen vor dem Harness — so testet jeder Spec gegen einen vollstaendigen Produktivstand.

  • Der Sankey-Klick ist der harte Teil.

    ECharts-Instanzen liegen in einer Closure, click ist ein Maus-Event. Ein minimaler Test-Seam (window.__sankeyDrill) macht den Drill-down deterministisch — ohne Verhaltensaenderung.

  • Hermetisch durch Vendoring.

    ECharts wandert vom CDN nach web/vendor/echarts/ — netzunabhaengige Tests und offline-faehiges Deployment.

  • Anlassfall im Plan verankert.

    Task 6 reproduziert den gemeldeten Sankey-Fehler; ist es ein echter Bug, wird er im selben Zug in dashboard.js behoben.

03Decisions

Key decisions & options

Test-Tool

Playwright Test

Eine Abhaengigkeit deckt Runner, Browser, webServer-Orchestrierung, setInputFiles und Auto-Retry-Assertions ab. Recherche hat Playwright 1.60 + Chromium im Container real gestartet.

Puppeteer/WebdriverIO — brauchen separaten Runner und eigene Server-Lifecycle-Loesung.

Cypress — In-Page-Modell vertraegt sich schlecht mit WASM-Workern.

Sankey-Klick deterministisch testen

Produktiv-Test-Seam

Handler-Rumpf in drillAufKnoten(d) faktorisieren, window.__sankeyDrill(name) exponieren — 3–5 Zeilen, keine Verhaltensaenderung. Isoliert die Drill-Logik von Render-/Timing-Effekten.

Reiner Pixel-Klick auf den Canvas — sproede; bleibt nur als zusaetzlicher Smoke-Test.

Versionsstempel-Mechanik

version.json + fetch

pages.yml schreibt web/version.json mit GITHUB_SHA; ein committeter dev-Fallback haelt den lokalen Betrieb sauber. web/ bleibt im Git byte-identisch.

Platzhalter-sed in index.html — haesslicher Platzhalter im committeten Quelltext, schlechtere lokale DX.

CI-Anbindung

Separater Workflow

Neuer .github/workflows/e2e.yml auf ubuntu-latest mit playwright install --with-deps chromium — getrennt von pages.yml, kein e2e-Job im Deploy.
