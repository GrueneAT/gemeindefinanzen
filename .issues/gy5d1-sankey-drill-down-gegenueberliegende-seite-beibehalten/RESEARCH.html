Strategy summary

Eng begrenzter Ein-Datei-Bugfix in der reinen Funktion buildSankeyOption (web/js/sankey-drill.js). PR #3 fuehrte zwei aeussere Guards ein, die im Drill-down die gesamte Gegenseite ueberspringen — genau diese Guards muessen weg.

Die Korrektur ist mechanisch: beide Guards entfernen, den inneren else if (!expand) auf eine pro-Seite berechnete Flagge umstellen (quelleGedrillt / gruppeGedrillt). Der aufgeklappte-Zweig-Code bleibt unveraendert; nicht-gewaehlte Knoten der gedrillten Seite bleiben korrekt ausgeblendet, die Gegenseite wird wieder als Uebersicht gerendert. drillAufKnoten braucht keine Aenderung — der Handler ersetzt eine bestehende Ausklappung bereits.

  • Guards entfernen, Bedingung weiten — nicht oeffnen.

    Die Flagge ist seitenspezifisch. Ein blankes else wuerde die von PR #3 korrekt entfernten anderen Knoten derselben Seite wieder einblenden.

  • Zwei Test-Assertions kodieren das alte Verhalten.

    run.mjs Zeilen 352-360 und 397-405 fordern "keine Gegenseite" — sie schlagen nach dem Fix fehl und muessen umgeschrieben werden.

  • Kein Handler-, Kapp- oder Aggregations-Eingriff.

    drillAufKnoten, TOP_N, kappen und der Reset bleiben ausdruecklich unberuehrt.

02Flow

Task flow & dependencies

Step 1 buildSankeyOption korrigieren Beide Guards entfernen, else-if auf quelleGedrillt / gruppeGedrillt umstellen, Kommentare anpassen.
Step 2 Unit-Tests in run.mjs anpassen Zwei "nur gewaehlter Zweig"-Assertions umschreiben, neue Assertions fuer die Gegenseite in Uebersichtsform.
Step 3 e2e-Test praezisieren Kommentar in sankey.spec.mjs korrigieren, optional Assertion fuer sichtbare Gegenseite ergaenzen.
Step 4 Tests gruen npm run test:js und npm run test:e2e laufen lassen.

03Decisions

Key decisions & options

Wie die Gegenseite im Drill-down behandelt wird

Chosen

Aeussere Guards entfernen; die Gegenseite wird ueber den bestehenden Uebersicht-Render-Pfad gerendert — kein duplizierter Code. Knoten behalten drillExpandbar: true und bleiben voll interaktiv.

Eine zweite Render-Kopie nur fuer die Gegenseite — unnoetige Duplikation des Uebersicht-Pfads.

Bedingung fuer den Uebersicht-Zweig

Chosen

Pro Seite eine Boolean-Flagge (quelleGedrillt / gruppeGedrillt); der innere Zweig wird else if (!quelleGedrillt). Nicht-gewaehlte Knoten der gedrillten Seite landen mit false im Zweig — bleiben korrekt ausgeblendet.

Ein blankes else ohne Bedingung — wuerde die anderen Knoten der gedrillten Seite wieder einblenden und das von PR #3 gewollte Verhalten zerstoeren.

drillAufKnoten anfassen?

Out of scope

Nein. Verifiziert: Gegenseiten-Knoten werden mit expandbar: true gerendert, ein Klick greift den else if (d.drillExpandbar)-Zweig und ersetzt die bestehende Ausklappung — symmetrisch, ohne Handler-Aenderung.

Betragstreue-Tests (gruppenSumme / quellenSumme)

Verify

Bleiben voraussichtlich gruen: die Summen filtern nach Link-Richtung (source bzw. target === "Gemeindehaushalt"), die Gegenseite erzeugt nur Links der anderen Richtung. Nach der Aenderung dennoch per Testlauf bestaetigen.
