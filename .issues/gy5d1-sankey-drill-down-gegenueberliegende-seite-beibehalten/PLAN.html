Strategy summary

PR #3 hat den Sankey-Drill-down so geschnitten, dass beim Aufklappen eines Knotens die gesamte gegenueberliegende Seite verschwindet. Die Richtung: zwei aeussere Guards entfernen und die innere Uebersicht-Bedingung seitenspezifisch weiten — minimal, rein additiv, ohne Render-Pfad zu duplizieren.

  • Drei Render-Zustaende statt zwei.

    Jede Seite kennt: Uebersicht, eigene Seite gedrillt, Gegenseite. Die Gegenseite soll wie die Uebersicht rendern.

  • Bestehenden Zweig weiten, nicht duplizieren.

    Der else if (!expand)-Zweig wird ueber eine pro-Seite-Flagge zu else if (!<seite>Gedrillt).

  • Bedingung MUSS seitenspezifisch sein.

    Ein else { ... } wuerde die von PR #3 korrekt entfernten anderen Knoten der gedrillten Seite wieder einblenden — der einzige Fallstrick.

  • Kein Handler-Eingriff.

    drillAufKnoten in dashboard.js ersetzt eine bestehende Ausklappung bereits — Gegenseiten-Knoten behalten drillExpandbar.

03Decisions

Key decisions & options

Wie die Gegenseite im Drill-down gerendert wird

Gewaehlt

Beide aeusseren Guards entfernen und den bestehenden else if (!expand)-Zweig ueber eine pro-Seite-Flagge zu else if (!<seite>Gedrillt) weiten — die Gegenseite faellt damit in den vorhandenen Uebersicht-Pfad.

Den Uebersicht-Render-Pfad fuer die Gegenseite duplizieren — doppelter Code, Wartungslast, kein Vorteil.

Klick-Handler drillAufKnoten anfassen

Ausser Scope

Keine Aenderung. RESEARCH.md verifiziert: der Handler ersetzt eine bestehende Ausklappung bereits; Gegenseiten-Knoten behalten ueber den Uebersicht-Pfad drillExpandbar: true — ein Klick startet symmetrisch einen neuen Drill-down.

Bedingung zu else { ... } aufweiten

Verworfen

Ein unbedingtes else wuerde die anderen, nicht-gewaehlten Knoten der GEDRILLTEN Seite wieder einblenden — exakt das von PR #3 korrekt entfernte Verhalten. Die Flagge muss seitenspezifisch bleiben.
